Appendix Results Package
========================

Generated by: 09_appendix_package.ipynb

Purpose
-------
This package collects existing output files from the synthetic-control,
topology-augmented synthetic-control, TDA diagnostic, baseline placebo,
and leave-one-out robustness workflows.

This notebook does not re-estimate any model. It only copies existing
outputs, builds section-level indexes, creates a master manifest, writes
summary files, and optionally compresses the package into a zip archive.

Recommended Reading Order
-------------------------
1. Start with H_main_results_package for the paper-facing tables and figures.
2. Use A_authorweight_scm and C_baseline_scm to verify SCM construction.
3. Use B_authorweight_tda and D_baseline_tda to inspect topological diagnostics.
4. Use E_baseline_placebo for formal baseline placebo inference.
5. Use F_topology_augmented_comparison for topology-augmented SCM outputs.
6. Use G_topology_augmented_loo for leave-one-out robustness checks.

Leave-One-Out Robustness Note
-----------------------------
The folder G_topology_augmented_loo contains the topology-augmented SCM
leave-one-out robustness outputs. It should be treated as a robustness
module, not as the main estimation output. Failed or non-converged runs,
if present, are intentionally retained and indexed rather than silently
removed.

Baseline Placebo Note
---------------------
The package retains baseline placebo inference outputs because they are
part of the formal comparison and diagnostic workflow. Author-weight
placebo and topology-augmented placebo folders are excluded by design.

File Handling Rules
-------------------
- Hidden folders and .ipynb_checkpoints are ignored.
- Only files with these suffixes are copied: ['.csv', '.json', '.pdf', '.txt']
- File names are preserved unless duplicates require a unique suffix.
- If two files have the same name, the copied duplicate receives a suffix
  based on its parent folder name.
